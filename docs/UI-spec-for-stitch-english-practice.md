# 英语练习网页 UI 说明文档

## 1. 文档目标

本文件用于指导 Stitch 生成英语练习网页的 UI 界面。

目标是生成一个：

- 简洁明了
- 学习工具感强
- 首屏即可使用
- 桌面端优先，同时兼顾移动端

的单页 Web 界面。

## 2. 页面定位

这是一个英语阅读与听力练习页面。

用户可以：

- 从预设任务中选择一段英文
- 点击按钮播放整段语音
- 鼠标移入单词查看中文释义

整体感觉应偏“轻量学习工具”，不是复杂平台，也不是内容社区。

## 3. 设计关键词

- Clean
- Focused
- Educational
- Minimal
- Calm
- Readable
- Lightweight

避免：

- 复杂仪表盘感
- 过多装饰
- 强营销风格
- 花哨卡片堆叠
- 重运营后台风格

## 4. 整体视觉方向

### 4.1 风格

- 清爽、现代、克制
- 以阅读体验为核心
- 界面应给人“安静练习”的感觉

### 4.2 色彩

建议采用浅色背景。

推荐方向：

- 页面背景：暖白、浅米白、极浅灰蓝均可
- 主色：偏蓝或偏青绿，体现学习、清晰、可靠
- 强调色：用于播放按钮、选中任务、hover 状态
- 正文文字：深灰而非纯黑
- 中文释义 tooltip：高对比但不刺眼

避免：

- 大面积高饱和颜色
- 紫色主导风格
- 纯黑纯白强对比造成阅读疲劳

### 4.3 气质

- 不要太“儿童化”
- 不要太“企业后台化”
- 更像高质量在线学习工具

## 5. 页面结构

整个页面为单页结构。

推荐采用两栏布局，在桌面端展示为：

- 左侧：任务列表
- 右侧：当前任务内容与播放区域

在移动端改为上下结构：

- 上方：任务列表横向或纵向紧凑展示
- 下方：正文内容与播放按钮

## 6. 页面模块

## 6.1 顶部头部区域

需要包含：

- 页面标题：`English Practice`
- 一句简短说明：`Click a task to listen and move over words to see Chinese meanings.`

设计要求：

- 头部不要太高
- 信息简洁
- 可以有轻微品牌感，但不要像营销落地页

## 6.2 任务列表区域

用于展示预设的英语任务。

每个任务项建议包含：

- 编号
- 标题
- 难度标签
- 一行简短预览或说明

设计要求：

- 列表清晰，便于快速扫描
- 当前选中的任务有明显高亮
- hover 时有轻微反馈
- 不需要复杂插画或大缩略图

任务项视觉建议：

- 圆角卡片或轻量列表块
- 选中态使用主色边框、浅色背景或阴影
- 难度标签使用简洁 pill 样式

## 6.3 当前任务信息区域

位于正文上方，用于说明当前练习的上下文。

建议包含：

- 当前任务标题
- 难度标签
- 可选的简短描述

这一区域应简短，不要抢正文的视觉焦点。

## 6.4 播放控制区域

这是核心操作区之一。

建议包含：

- 主播放按钮
- 当前播放状态文案
- 可选的重播按钮

设计要求：

- 按钮要足够明显
- 第一眼能知道“这里可以播放”
- 播放中、暂停、结束三种状态要清楚

视觉建议：

- 播放按钮可用实心主色按钮
- 暂停与重播保持统一风格
- 不需要复杂进度条也可接受，但如果有进度反馈要尽量简洁

## 6.5 英文正文阅读区域

这是页面最重要的视觉中心。

要求：

- 大段英文可舒适阅读
- 行高宽松
- 字号适中
- 宽度控制在易读范围
- 视觉上有足够留白

正文应占据右侧主要空间，明显大于其他模块。

推荐感觉：

- 像高质量阅读卡片
- 有轻微容器感，但不要太厚重
- 可以有柔和边框或非常浅的阴影

## 6.6 单词释义 tooltip

这是本页面最有辨识度的交互点。

tooltip 需要在鼠标移入单词时显示。

tooltip 内容建议包含：

- 英文单词
- 中文释义
- 可选词性

设计要求：

- tooltip 小巧清晰
- 层级高于正文
- 不要遮挡太多阅读内容
- 出现速度快
- 样式统一精致

视觉建议：

- 深色底浅色字，或浅色底深色字都可以
- 圆角适中
- 有轻微阴影
- 可带小箭头，也可不带

## 7. 核心交互要求

## 7.1 默认状态

- 页面打开后默认选中第一个任务
- 右侧直接显示该任务正文
- 播放按钮处于可点击状态

用户不需要做额外初始化操作。

## 7.2 任务切换

- 点击左侧任务后，右侧内容立即切换
- 选中态同步变化
- 如果旧任务正在播放，切换时应停止旧音频

## 7.3 播放交互

- 点击播放后开始朗读
- 播放按钮切换为暂停或播放中状态
- 播放结束后恢复可再次播放状态

## 7.4 单词 hover

- 鼠标移入单词立即显示中文释义
- 鼠标移出后 tooltip 消失
- 支持仅对已配置释义的单词显示交互反馈

## 7.5 移动端兼容

移动端没有 hover，因此应改为：

- 点击单词显示释义
- 点击其他单词切换释义
- 点击空白区域关闭释义

## 8. 组件状态要求

Stitch 生成时请明确以下状态：

- 任务项默认态
- 任务项 hover 态
- 任务项选中态
- 播放按钮默认态
- 播放按钮 hover 态
- 播放中状态
- 单词默认态
- 单词可释义态
- tooltip 显示态
- 移动端点击释义态

## 9. 排版要求

- 页面有明确留白
- 不要把元素挤得过满
- 英文正文是最大的阅读焦点
- 左侧列表宽度适中，不抢正文空间
- 标题、副标题、正文、辅助说明要有清晰字号层级

建议的阅读体验：

- 页面宽而不空
- 组件之间呼吸感明显
- 正文区域不要超过舒适阅读宽度

## 10. 字体建议

如果 Stitch 支持字体建议，优先考虑：

- 英文标题与正文：清晰、现代、适合阅读的字体
- 中文 tooltip：简洁易读的无衬线字体

整体不要使用过于艺术化或装饰性的字体。

## 11. 动效建议

动效要少而精。

建议仅保留：

- 任务切换的轻微状态过渡
- 按钮 hover 反馈
- tooltip 出现/消失的快速淡入淡出

避免：

- 大幅弹跳动画
- 复杂滚动动效
- 多层级卡片飞入

## 12. 响应式要求

### 桌面端

- 采用左右分栏
- 左窄右宽
- 正文区域为绝对主视觉

### 平板端

- 可保留分栏，但适当缩小左右间距

### 手机端

- 改为上下结构
- 播放按钮保持醒目
- 单词点击释义必须易用

## 13. 不希望出现的设计

- 暗黑风主页面
- 游戏化过强的奖励式界面
- 复杂侧边导航
- 多层 tab
- 浓重拟物风
- 过度渐变
- 花哨插画占据主区域
- 看起来像销售落地页

## 14. 可直接给 Stitch 的生成指令

请生成一个简洁、现代、偏学习工具风格的英语练习单页 Web UI。

页面用于英语段落练习，桌面端优先，移动端兼容。页面采用浅色背景、清晰留白和安静专注的阅读氛围。整体布局为两栏结构：左侧是预设练习任务列表，右侧是当前任务详情、播放按钮和英文正文阅读区。英文正文是页面核心视觉焦点，要求大段文字舒适易读。

左侧任务列表中的每个任务项包含编号、标题、难度标签和一句简短预览。选中态要明显，hover 有轻微反馈。右侧顶部展示当前任务标题和难度，下面是醒目的播放按钮区域，再下面是主要的英文段落内容。

正文中的可释义单词需要具备 hover 提示感，鼠标移入时显示精致的小型 tooltip，内容为英文单词和中文释义。整体交互要非常直观，不要复杂流程，不要仪表盘风格，不要营销感。请使用现代教育产品的视觉语言，简洁、克制、专注、可读。

## 15. 交付要求

Stitch 产出的 UI 稿应至少体现：

- 桌面端完整页面
- 移动端适配思路
- 任务列表选中态
- 播放按钮状态
- 英文正文区域
- 单词 tooltip 展示效果

如果支持多状态输出，优先补充：

- 默认页面
- 任务切换后页面
- 单词 hover tooltip 页面状态
